# Claude Code × ServiceNow SDK — Demo App

This repository demonstrates how to use **Claude Code** with the **ServiceNow SDK (now-sdk)** to build production-ready ServiceNow applications from natural language — no manual metadata authoring required.

The Market Research Email Parser is the sample app used throughout the demo.

---

## What Claude Code Built (the demo app)

An end-to-end ServiceNow application that:

1. **Triggers** on any inbound email with `Market Research` in the subject
2. **Archives** the raw email in a custom `Email Data` table
3. **Creates a seed record** in the `Market Research` table
4. **Fires a Business Rule** (after insert) that parses the HTML body — extracting every `<tr>/<td>` from the competitor table
5. **Expands** the seed into one structured record per competitor, with a recursion guard (`is_parsed`) so the BR never double-fires
6. **Surfaces** the data through a custom app navigator menu

All of this was generated by Claude Code in a single conversation using the `sndev` skill.

## Using the Claude Code Skill

### 1. Install the skill

Copy `SKILL.md` to your Claude Code skills directory:

```bash
cp "Tutorial instructions and skills/SKILL.md" ~/.claude/skills/sndev.md
```

### 2. Open your ServiceNow SDK project with Claude Code

```bash
cd my-servicenow-app
claude
```

### 3. Ask Claude to build what you need

The skill triggers automatically whenever you describe a ServiceNow task. Examples:

```
"Create an inbound email flow that fires when subject contains Market Research"

"Add a Business Rule that parses the HTML body and creates one record per table row"

"Build a table to store competitor data with fields for name, segment, pricing, market share, NPS, and growth"

"Add a UI policy to hide the source email field from the form"

"Create an application menu with modules for Market Research Data and Email Archive"
```

Claude Code will scaffold the correct Fluent TypeScript, fix any SDK type errors, and run `now-sdk build` and `now-sdk install` — all from the conversation.

## Demo App Source

The generated source code for the Market Research Email Parser lives in [`src/fluent/`](./src/fluent/):

```
src/fluent/
├── tables/
│   ├── email-data.now.ts            # Email archive table
│   └── market-research.now.ts       # Competitor data table (7 fields + recursion guard)
├── flows/
│   └── inbound-email-parser.now.ts  # Inbound Email Flow
├── business-rules/
│   ├── market-research-parser.now.ts
│   └── market-research-parser.server.js   # HTML <tr>/<td> parser
├── navigation/
│   └── application-menu.now.ts      # App menu + List modules
├── ui-policies/
│   └── market-research-ui-policies.now.ts # Hides internal fields from form
└── index.now.ts
```
